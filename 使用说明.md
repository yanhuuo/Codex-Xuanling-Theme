# 秧秧·玄翎｜苍羽夜 Codex Dream Skin

这是为 Codex Desktop 制作的深色主题包：人物位于右侧，左侧保留代码与对话阅读区；配色以玄翎黑、苍羽青和少量湮灭紫为主。

## 安装

1. 双击 `安装主题工具.cmd`。
2. 如果 Codex 正在运行，安装器会提示确认重启，避免未经确认丢失输入。
3. 工具会复制到 `%LOCALAPPDATA%\CodexDreamSkin\runtime`，并创建桌面 `Codex 主题` 入口。
4. 以后拿到新版安装包，重新双击即可原位升级；Store 更新导致注入器失效时，后台守护会自动匹配最新的已注册 Codex 包并修复。

旧的 `安装玄翎主题.cmd` 仍可使用，但建议统一使用新的主题工具安装器。

也可以在 PowerShell 中运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\install-xuanling.ps1
```

在设置的“主题”页中可以选择主题宠物。选择会把标准 v2 宠物完整复制到主题文件夹；通过 GitHub 或本地主题库换机后，启用主题会自动安装并选中同一只宠物。玄翎主题默认携带当前选择的 `尤诺（iuno）`。

## 主题页

打开 Codex 的 `设置 → 主题`。这是独立渲染的主题管理页，只依赖设置导航的语义标记，不嵌入“外观”等原生设置内容，因此常规版本更新不需要跟着内部布局逐项调整。

在这里可以：

- 切换已安装主题；
- 保存主题文件后自动热重载，不重启 Codex；
- 一键“还原官方外观”，主题和主题库仍会保留；
- 再次启用当前主题；
- 添加本地主题文件夹；
- 直接添加公开的 GitHub 主题仓库；
- 添加 HTTPS 远程主题库索引并安装其中的主题。
- 读取 `~/.codex/pets` 中的原生 v2 宠物，并为主题记录配套宠物。

每个主题现在都是独立代码包，目录结构为：

```text
主题文件夹/
├─ theme.json
├─ theme.css
├─ theme.js
└─ background.jpg
```

图标、动画和页面适配应写在该主题自己的 `theme.css/theme.js` 中，不再放进公共注入器。完整格式见 `windows/THEME_FORMAT.md`。

远程索引的最小格式：

```json
{ "themes": [{ "id": "theme-id", "name": "主题名", "themeUrl": "https://example.com/theme.json" }] }
```

GitHub 仓库安装使用仓库根目录的 `theme-library.json`。主题页只接受公开 HTTPS GitHub 仓库；私有仓库不会要求或保存 GitHub Token。

## 热重载

注入器会监听当前活动主题的 `theme.json`、`theme.css`、`theme.js` 和图片文件。文件保存并稳定约 120 ms 后会重新校验和注入；无效修改会被拒绝，当前可用主题继续保留。文件系统监听不可用时自动退回定时校验。

## 宠物关联

宠物关联不会修改 Codex 原生宠物包。主题页只读取并验证 `~/.codex/pets/<id>/pet.json` 和 `spritesheet.webp`，仅显示 `spriteVersionNumber: 2` 的有效包。关联关系保存在 `%LOCALAPPDATA%\CodexDreamSkin\pet-associations.json`，宠物显示和动画仍由 Codex 原生宠物窗口负责。

## 还原与卸载

设置页的“还原官方外观”只暂停视觉主题，不删除已安装主题，适合日常切换。

桌面的 `Codex Dream Skin - Restore` 用于完整恢复：关闭主题会话并恢复基础配置。安装器会把原配置备份到 `%LOCALAPPDATA%\CodexDreamSkin`。

## 安全边界

- 仅使用 `127.0.0.1` 上的本机 CDP 端口。
- 不修改 `WindowsApps`、`app.asar` 或 Codex 安装文件。
- 自动修复只识别已注册且签名类型为 Store 的 `OpenAI.Codex` 包，并继续只使用回环 CDP。
- 壁纸是为本主题生成的原创同人视觉，不是官方宣传图。

底层注入方案来自 Fei-Away/Codex-Dream-Skin；原仓库说明保留在同目录的 `README.md`。
