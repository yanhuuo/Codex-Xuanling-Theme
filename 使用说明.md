# 秧秧·玄翎｜苍羽夜 Codex Dream Skin

这是为 Codex Desktop 制作的深色主题包：人物位于右侧，左侧保留代码与对话阅读区；配色以玄翎黑、苍羽青和少量湮灭紫为主。

## 安装

1. 双击 `安装主题工具.cmd`。
2. 如果 Codex 正在运行，安装器会提示确认重启，避免未经确认丢失输入。
3. 工具会复制到 `%LOCALAPPDATA%\CodexDreamSkin\runtime`，并创建桌面 `Codex 主题` 入口。
4. 首次启动只出现 `设置 → 主题` 管理页，并保持 Codex 官方外观。
5. 在主题页的“可安装主题”中点击“安装主题”，再从“已安装主题”启用玄翎或其他主题。
6. 以后拿到新版安装包，重新双击即可原位升级。后台守护只会在已存在且已验证的 CDP 会话中修复注入器，绝不会自行重启普通 Codex；连续失败会自动熔断并等待手动启动。

旧的 `安装玄翎主题.cmd` 只用于兼容；建议只安装主题管理工具，再在主题页完成主题安装。

也可以在 PowerShell 中运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\install-xuanling.ps1
```

在设置的“主题”页中可以选择主题宠物。主题只记录宠物 ID，宠物包统一保存在项目根目录 `pets/`；通过 GitHub 或本地主题库换机后，启用主题会自动安装并选中同一只宠物。玄翎主题默认绑定 `秧秧·玄翎（官鼓连续版 R3）`。

## 主题页

打开 Codex 的 `设置 → 主题`。这是独立渲染的主题管理页，只依赖设置导航的语义标记，不嵌入“外观”等原生设置内容，因此常规版本更新不需要跟着内部布局逐项调整。

在这里可以：

- 切换已安装主题；
- 安装安装包内置主题；
- 保存主题文件后自动热重载，不重启 Codex；
- 一键“还原官方外观”，主题和主题库仍会保留；
- 再次启用当前主题；
- 添加本地主题文件夹；
- 直接添加公开的 GitHub 主题仓库；
- 添加 HTTPS 远程主题库索引并安装其中的主题。
- 读取 `~/.codex/pets` 中的原生 v2 宠物，并把所选宠物作为独立 `pets/` 包绑定到主题。
- 从内置主题复制布局，选择本机背景和可选 `icons.json`，另存为不会上传的本地主题。

每个主题现在都是独立配置包，公共布局与注入逻辑只维护一份：

```text
主题文件夹/
├─ install.json
├─ theme.json
├─ theme.css
├─ icons.json
└─ background.jpg

windows/engine/
├─ theme-base.css
└─ theme-runtime.js

项目根目录/pets/<pet-id>/
├─ pet.json
└─ spritesheet.webp
```

主题自己的 `install.json`、`theme.json`、`theme.css` 和 `icons.json` 管理色板、背景、品牌图标与资源；设置页适配、动画和 DOM 行为由公共框架统一提供。完整格式见 `windows/THEME_FORMAT.md`。

点击主题列表中的“本地自定义主题”卡片会打开分区设置弹窗。背景通过文件选择器选择；基础主题的全部图标以独立表单项和真实图标预览显示，可逐项选择 SVG，也可导入只包含部分覆盖项的 `icons.json` 并即时预览，其余图标继承基础主题。生成主题位于 `%LOCALAPPDATA%\CodexDreamSkin\themes`，卡片会显示“仅本地”，不会进入 GitHub 主题索引。

远程索引的最小格式：

```json
{ "themes": [{ "id": "theme-id", "name": "主题名", "themeUrl": "https://example.com/theme.json" }] }
```

GitHub 仓库安装使用仓库根目录的 `theme-library.json`。主题页只接受公开 HTTPS GitHub 仓库；私有仓库不会要求或保存 GitHub Token。

## 热重载

注入器会监听当前活动主题的 `install.json`、`theme.json`、`theme.css`、`theme.js`、`icons.json` 和图片文件。文件保存并稳定约 120 ms 后会重新校验和注入；无效修改会被拒绝，当前可用主题继续保留。文件系统监听不可用时自动退回定时校验。

## 主题宠物

主题页只读取并验证 `~/.codex/pets/<id>/pet.json` 和 `spritesheet.webp`，仅显示标准 v2 有效包。仓库包位于项目根目录 `pets/<id>/`，主题目录只绑定 ID；换机安装并启用主题时，会同步安装和选择该宠物。宠物显示和动画仍由 Codex 原生宠物窗口负责。

## 还原与卸载

设置页的“还原官方外观”会暂停视觉主题并恢复安装前的基础外观配置，但不删除主题、宠物和主题库。

桌面的 `Codex Dream Skin - Restore` 用于完整恢复：关闭主题会话并恢复基础配置。安装器会把原配置备份到 `%LOCALAPPDATA%\CodexDreamSkin`。

## 安全边界

- 仅使用 `127.0.0.1` 上的本机 CDP 端口。
- 不修改 `WindowsApps`、`app.asar` 或 Codex 安装文件。
- 自动修复只识别已注册且签名类型为 Store 的 `OpenAI.Codex` 包，并继续只使用回环 CDP；它不会强制重启 Codex，连续失败会写入 `guard.failed.json` 并停用。
- 壁纸是为本主题生成的原创同人视觉，不是官方宣传图。

底层注入方案来自 Fei-Away/Codex-Dream-Skin；原仓库说明保留在同目录的 `README.md`。
